# Definición funcionalidades vistas
## Definición de meta funcionalidad
Un proceso es un conjunto de tareas que deben completarse para que se cumpla un objetivo. En SETAP solo nos avocaremos a la definición del conjunto de tareas que tendrá asociado cada proceso y además que este estará indexado a un proveedor en especifico. En términos prácticos un proceso en SETAP es un conjunto de tareas que esta agrupado por un id de proceso, que a su vez esta asociado a un proveedor_id único. Así, un conjunto de tareas que tiene en común un id de proceso, conforma un proceso en SETAP.

### create.php
Esta funcionalidad tendrá las siguientes facilidades uso:
- un selector de proveedor que deberá ser visible cuando el usuario es un admin (usuario_id = 1)
- un selector de tareas categorías
- un selector de tareas, filtradas por proveedor seleccionado y por categoría seleccionada, con buscador.
- un control numérico para indicar duración en horas, mínimo 0.5, step 0.5
- un botón para levantar un pop-up (modal) para crear una tarea nueva, que debe tener lo necesario como en la vista "\comercial-elroblev2\setap\src\App\Views\tasks\newTask.php", al grabar tarea nueva debe refrescar el selector de tareas.
- un botón agregar tarea al proceso
- un botón limpiar, que elimina todas las tareas asociadas a un proceso
- una tabla que muestre las tareas que se van agregando al proceso. Luego en esta tabla se debe tener las siguientes columnas
-- Tarea (nombre)
-- Duración
-- Categoría
-- Acciones, aquí se debe tener los siguientes botones:
--- Ver, para levantar pop-up (modal) que muestra los datos de la tarea en la línea
--- Eliminar, para eliminar proceso la tarea seleccionada

### edit.php
Esta funcionalidad tendrá lo necesario para editar un proceso:
- un selector de proveedor que deberá ser visible cuando el usuario es un admin (usuario_id = 1)
- un selector de procesos, que se seleccionara automáticamente cuando el edit.php haya sido invocado desde list.php, con buscador.
- Todos los controles el create.php

### view.php
Esta funcionalidad tendrá un lo necesario solo para ver de que tareas esta compuesto un procesos
- un selector de proveedor que deberá ser visible cuando el usuario es un admin (usuario_id = 1)
- un selector de procesos, que se seleccionara automáticamente cuando el edit.php haya sido invocado desde list.php, con buscador.
- una tabla que muestre las tareas que se van agregando al proceso. Luego en esta tabla se debe tener las siguientes columnas
-- Tarea (nombre)
-- Duración
-- Categoría
-- Acciones, aquí se debe tener los siguientes botones:
--- Ver, para levantar pop-up (modal) que muestra los datos de la tarea en la línea

### list.php
Esta funcionalidad tendrá lo necesario para ver los procesos que tiene creados un proveedor determinado
- un selector de proveedor que deberá ser visible cuando el usuario es un admin (usuario_id = 1)
- Botón crear proceso, invoca create.php
- Tabla con lista de procesos, columnas
-- Nombre
-- Descripción, hasta 100 caracteres de la descripción
-- Acciones
--- Ver, invoca el view.php
--- Editar, invoca el edit.php
--- Eliminar, elimina proceso
